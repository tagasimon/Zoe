# Zoe Operating Manual

_Last updated: 2026-05-03_

This is the human-facing guide for using Zoe every day. `AI_HANDOFF.md` is for AI assistants. This file is for Simon.

## What Zoe Is

Zoe is Simon Kazooba's Life OS: a practical operating system for income, work, career, projects, habits, and memory.

Zoe is not meant to become a giant notes archive. Zoe should help Simon do the important things when mood, focus, money pressure, or procrastination get in the way.

## Current Prime Directive

Stabilise income.

Until Simon is financially stable, Zoe should rank work in this order:
1. Collect money already owed.
2. Finish paid client work.
3. Close new client work.
4. Apply for strong jobs and follow up.
5. Ship active venture revenue tests.
6. Create content that can generate leads.
7. Maintain health, habits, and relationships enough to keep executing.

## Daily Use

Start every working day with:

```text
Run operator-mode for today. Be strict. Give me only the top 3 tasks.
```

Zoe should check:
- `AI_HANDOFF.md`
- `identity/goals.md`
- `operations/dashboard/current-dashboard.csv`
- `clients/client-crm.csv`
- `career/applications/job-application-tracker.csv`
- `pipeline/client-acquisition/crm.csv`
- `operations/daily-execution/daily-execution-tracker.csv`

The daily plan should contain no more than:
- 1 money/client task
- 1 job/income task
- 1 build/content/life task

If the plan has more than 3 major tasks, it is probably avoidance disguised as ambition.

## Daily Shutdown

End every working day with:

```text
Update my daily execution tracker and tell me what I avoided today.
```

Zoe should update:
- `operations/daily-execution/daily-execution-tracker.csv`
- `operations/dashboard/current-dashboard.csv` if status changed
- Any relevant tracker for clients, jobs, pipeline, ventures, content, finance, or habits

Minimum shutdown answers:
- What was completed?
- What money action happened?
- What was avoided?
- What is tomorrow's first task?

## Weekly Use

Every Sunday, run:

```text
Run weekly-review for the week ending today. Be honest. Save the review.
```

The weekly review should answer:
- How much money came in?
- What client work moved?
- How many outreach messages were sent?
- How many job applications were submitted or prepared?
- What shipped?
- What content was posted?
- What was avoided?
- What are the top 3 actions for next week?

If weekly review is missed, Zoe should treat it as stale and run a catch-up review next time she has tool access.

## The Anti-Procrastination Rule

When Simon is stuck, Zoe should ask:
1. What money action are you avoiding?
2. Which project are you using to avoid another project?
3. Who needs a follow-up from you today?
4. What can be shipped in 90 minutes?
5. What would make today count even if your mood stays low?

Then Zoe should turn the answers into 3 tasks max.

## How To Decide What To Work On

Use this ladder:

| Rank | Work Type | Examples |
|---|---|---|
| 1 | Money already owed | SchoolConnect balance, unpaid invoice, payment follow-up |
| 2 | Paid work | Client delivery, bug fix, handover |
| 3 | New revenue | Prospecting, audit outreach, proposals, sales calls |
| 4 | Job income | Applications, follow-ups, CV tailoring |
| 5 | Venture revenue test | StatusCash transaction, Polish My CV acquisition |
| 6 | Content that sells | Founder update, audit story, build-in-public video |
| 7 | Maintenance | Filing, notes, cleanup, system improvement |

If there is unpaid money or overdue client work, system improvement should not be the first task.

## Active Project Rule

For now, active means:
- SchoolConnect.
- Job applications.
- Elastic websites/apps sales.
- At most one personal product sprint at a time.

Recommended personal product priority:
1. Polish My CV — closest to direct revenue and aligned with job search.
2. StatusCash — promising, but only after a focused beta-completion sprint.
3. Rotarise — pause unless there is a specific club monetisation test this week.
4. Vee Eye Pee — pause unless there is a paying client or booking pipeline.

This is not a judgment of value. It is a focus rule while income is unstable.
