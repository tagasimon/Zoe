# Deep Work Rules

## What Deep Work Is

A focused block where you produce something real: code shipped, proposal sent, client messaged, feature built.

It is not: reading, planning, "researching", scrolling, or thinking about what to do next.

---

## Block Structure

| Block | Time | Use for |
|-------|------|---------|
| Morning block | 08:00 – 11:00 | Hardest task. Client work, code, proposals. |
| Midday check | 11:00 – 11:15 | Email + messages only. |
| Afternoon block | 12:00 – 14:00 | Second task. Outreach, follow-ups, admin. |
| End-of-day | 17:00 – 17:15 | Update today.md, scoreboard, tomorrow's task 1. |

**Default: protect the morning block at all costs.**

---

## Rules During a Block

1. Phone face down, notifications off
2. One tab: the thing you are working on
3. If you get stuck — write what you're stuck on, then keep going
4. No checking email until the midday check
5. If something "urgent" appears — write it in the parking lot on today.md, do not switch tasks

---

## Signs You Are Not in Deep Work

- You have 5+ tabs open
- You're reading something "for background"
- You've been "thinking about the problem" for 20+ minutes without writing or coding
- You switched tasks without finishing the first one

If any of these are true: close everything, reopen today.md, pick task 1, start.

---

## Emergency Reset

When procrastination hits:
1. Set a 25-minute timer
2. Work on task 1 only — no perfection, just progress
3. When timer ends — take 5 minutes, then set another 25
4. Do this 3 times. That is your morning block.

_The first 2 minutes are always the hardest. Start before you feel ready._
